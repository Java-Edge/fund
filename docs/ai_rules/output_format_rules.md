# 输出格式规则

## 规则名称：证据先行的结构化输出

- 触发条件：所有分析输出。
- 证据要求：先给证据，再给判断，最后给建议。
- 输出限制：使用结构化 Markdown，避免大段无分层文本；不能输出确定性收益承诺、确定性买卖结论、绝对化措辞。
- 风险提示：如果证据不足，优先提醒证据有限，而不是把格式写得很满。

## 规则名称：标准分析的四段结构

- 触发条件：标准分析模式。
- 证据要求：四部分都要有对应的数据支撑。
- 输出限制：输出应覆盖市场趋势、板块机会、基金建议、风险提示四部分；每一部分都要有“现象 + 判断 + 建议/提示”。
- 风险提示：不要出现只有建议没有风险约束的段落。

## 规则名称：快速分析的短结构

- 触发条件：快速分析模式。
- 证据要求：每段至少引用一个最核心事实支撑判断。
- 输出限制：仅输出四段：市场趋势、板块机会、基金建议、风险提示。
- 风险提示：每段都必须包含一句风险或约束，不可只写乐观观点。

## 规则名称：深度研究的完整报告结构

- 触发条件：深度研究模式。
- 证据要求：报告必须覆盖市场趋势、板块机会、基金组合建议、风险提示、信息来源说明。
- 输出限制：网络搜索获得的信息应尽量保留 Markdown 链接引用；持仓基金分析必须结合基金表现与相关新闻共同展开。
- 风险提示：深度并不代表更激进，长报告也必须保留谨慎措辞。
